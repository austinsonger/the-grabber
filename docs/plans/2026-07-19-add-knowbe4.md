# Add KnowBe4 Evidence Provider Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add KnowBe4 (security awareness training / phishing simulation SaaS) as a first-class evidence-collection provider — a workspace crate for the KnowBe4 Reporting API plus eight collectors across users, groups, phishing simulation, and security awareness training — wired into the existing TUI/CLI plumbing exactly like Okta.

**Architecture:** Mirror the existing Okta integration exactly. A new `crates/knowbe4-rs` workspace crate wraps the KnowBe4 Reporting API (`Authorization: Bearer <api_key>` auth, `page`/`per_page=500` pagination, pure exponential-backoff retry on HTTP 429 — no rate-limit-reset header to honor, unlike Okta). A new `src/providers/knowbe4` module implements `ProviderFactory` and produces eight `CsvCollector` implementations — never `JsonCollector`/`EvidenceCollector`, matching every other non-AWS provider in this codebase. KnowBe4-specific config lives in `knowbe4-config.toml` (gitignored), merged into the main `AppConfig` at startup like `okta-config.toml`. The TUI gains a `Knowbe4` variant on `CloudProvider`; no endpoint- or project-selection screen is needed because every KnowBe4 collector operates account-wide, exactly like Okta's per-tenant domain. The one wrinkle beyond Okta's shape: KnowBe4's Reporting API is split across five regional hosts, so the client carries a `Knowbe4Region` concept Okta doesn't need.

**Tech Stack:** Rust 1.75+, `reqwest` (rustls), `serde`/`serde_json`, `tokio`, `async_trait`, `chrono`. Optional Cargo feature `knowbe4`, added to the `default` feature set alongside `tenable`/`okta`/`jira`/`elastic`/`github`.

**Reference patterns mirrored throughout this plan** (see `docs/plans/spec/2026-07-19-knowbe4-provider-design.md` for the full design rationale — this plan implements that spec as written):
- API client, retry, base struct shape: `crates/okta-rs/src/client.rs` (KnowBe4's retry has no rate-limit-reset header and its pagination is `page`/`per_page`, not `Link` headers — both call sites below are original to this plan, not ports)
- Provider factory + flat collector registration by string key: `src/providers/okta/factory.rs`
- Self-contained nested (per-parent fan-out) collector: `src/providers/okta/groups.rs` (`OktaGroupMembersCollector`)
- Per-item fan-out collector (one API call per user): `src/providers/okta/factors.rs`
- Client-side date-window filtering with a 90-day fallback: `src/providers/okta/system_log.rs` (`OktaSystemLogCollector`)
- Graceful 404-degrades-to-empty pattern: every Okta collector's `Err(OktaError::Api { status: 404, .. }) => return Ok(vec![])` arm
- TUI collector menu: `src/tui/menus/okta.rs` + `src/tui/menus/mod.rs`
- Account config fields + resolvers + config-file merge: `src/app_config.rs` Okta fields (lines 182–189), resolvers (lines 267–281), merge block (lines 394–402)
- TUI session account-prep block: `src/runner/tui_session.rs` Okta block (lines 728–836)
- Provider-switch nav wiring: `src/tui/app/nav.rs` (whole file; Okta/Jira/Elastic/Github arms at lines 107–114, 161–166, 240–287)
- Provider selection screen and key handler are already fully generic over `CloudProvider::available()` (`src/tui/ui/account_screens.rs::draw_provider_selection`, `src/tui/events.rs::handle_provider_selection`) — **no changes needed there**, verified by reading both files; a stale earlier GitHub plan lists them as modified but the codebase was generalized since (commit `919b552`).

**Out of scope:**
- KnowBe4 SCORM/store-purchase/policy-acknowledgment endpoints — only the eight collectors in the design spec's data-scope table.
- A region-picker TUI screen — region is config/env-driven only (`knowbe4_region` / `KNOWBE4_REGION`), like Tenable's `tenable_url`.
- Any interactive credential-entry TUI screen — KnowBe4 credentials come exclusively from `knowbe4-config.toml` / env vars.
- `wiremock` HTTP tests for `knowbe4-rs` — per the design spec's Testing section, this project's established convention for straightforward provider additions is `cargo check`/`clippy`/`fmt` verification plus a final manual smoke run, not a per-crate test suite.

## Global Constraints

- Rust 1.75+, edition 2021, matching every other crate in this workspace.
- `reqwest` with `rustls-tls`, `default-features = false` — matches `okta-rs`/`tenable-rs`/`jira-rs`/`github-rs`.
- Every collector is a `CsvCollector` (`src/evidence.rs`) — never `JsonCollector`/`EvidenceCollector`. `Knowbe4ProviderFactory::json_collectors()`/`evidence_collectors()` both return `Vec::new()`.
- Auth: `Authorization: Bearer <api_key>` on every request. No other headers required by the API beyond `Accept: application/json`.
- Pagination: `page`/`per_page=500` query params, looped until a page returns fewer than 500 items — **not** `Link`-header pagination.
- Retry: HTTP 429 only, pure exponential backoff (1s → 2s → 4s → 8s → 16s, capped at 30s), 5 attempts, no rate-limit-reset header parsing.
- 404 responses from list endpoints degrade to `Ok(vec![])` at the collector layer (never at the crate layer, which always propagates `Knowbe4Error::Api`). No `unwrap`/`expect` in production code — `anyhow::Result` / `anyhow::Context` throughout the provider layer.
- Time-windowed collectors (`knowbe4-risk-score-history`, `knowbe4-phishing-test-recipients`, `knowbe4-training-enrollments`) filter client-side on the resource's own timestamp field and default to the last 90 days when `dates` is `None`, mirroring `OktaSystemLogCollector`.
- `knowbe4_region_resolved()` on `Account` returns a plain `String` (not a `knowbe4_rs::Knowbe4Region`) — `src/app_config.rs` has no `#[cfg(feature = "knowbe4")]` gating today (every other provider's fields/resolvers there are unconditional), so it must never name a type from an *optional* dependency. `knowbe4_rs::Knowbe4Region` is only referenced from code already behind `#[cfg(feature = "knowbe4")]` (`src/providers/knowbe4/*`, the KnowBe4 block in `src/runner/tui_session.rs`). This keeps `cargo build --no-default-features --features tenable` (documented in `CLAUDE.md`) working.
- No `unsafe`. Imports grouped std → external crates → `crate::*`, blank line between groups (per `AGENTS.md` conventions referenced in `CLAUDE.md`).

use std::fmt;
use std::str::FromStr;

/// The five regional hosts KnowBe4's Reporting API is split across.
/// A KnowBe4 account is provisioned in exactly one region; requests to the
/// wrong region's host fail auth entirely, so this is a required client
/// input, not just a display label.
#[derive(Debug, Clone, Copy, PartialEq, Eq, Default)]
pub enum Knowbe4Region {
    #[default]
    Us,
    Eu,
    Ca,
    Uk,
    De,
}

impl Knowbe4Region {
    pub fn base_url(&self) -> String {
        format!("https://{}.api.knowbe4.com", self.host_segment())
    }

    fn host_segment(&self) -> &'static str {
        match self {
            Knowbe4Region::Us => "us",
            Knowbe4Region::Eu => "eu",
            Knowbe4Region::Ca => "ca",
            Knowbe4Region::Uk => "uk",
            Knowbe4Region::De => "de",
        }
    }
}

impl fmt::Display for Knowbe4Region {
    fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {
        write!(f, "{}", self.host_segment())
    }
}

impl FromStr for Knowbe4Region {
    type Err = ();

    fn from_str(s: &str) -> Result<Self, Self::Err> {
        match s.trim().to_lowercase().as_str() {
            "us" => Ok(Knowbe4Region::Us),
            "eu" => Ok(Knowbe4Region::Eu),
            "ca" => Ok(Knowbe4Region::Ca),
            "uk" => Ok(Knowbe4Region::Uk),
            "de" => Ok(Knowbe4Region::De),
            _ => Err(()),
        }
    }
}
```

const MAX_RETRIES: u32 = 5;
const MAX_BACKOFF_SECS: u64 = 30;
const PAGE_SIZE: usize = 500;

/// Async HTTP client for the KnowBe4 Reporting API.
///
/// Auth: `Authorization: Bearer <api_key>` is injected on every request.
/// Retries HTTP 429 with pure exponential backoff (1s→2s→4s→8s→16s, capped
/// at 30s) — KnowBe4's Reporting API documents no `X-Rate-Limit-Reset`-style
/// header to honor, unlike Okta.
///
/// `Knowbe4Client` is cheaply cloneable — `reqwest::Client` is arc-pooled.
#[derive(Clone)]
pub struct Knowbe4Client {
    pub(crate) http: Client,
    pub(crate) base_url: String,
}

impl Knowbe4Client {
    /// Build a client for the given KnowBe4 Reporting API region.
    pub fn new(region: Knowbe4Region, api_key: &str) -> Result<Self, Knowbe4Error> {
        let trimmed = api_key.trim();
        if trimmed.is_empty() {
            return Err(Knowbe4Error::InvalidApiKey);
        }
        let auth = format!("Bearer {trimmed}");
        let mut headers = header::HeaderMap::new();
        headers.insert(header::AUTHORIZATION, header::HeaderValue::from_str(&auth)?);
        headers.insert(
            header::ACCEPT,
            header::HeaderValue::from_static("application/json"),
        );

        let http = Client::builder().default_headers(headers).build()?;
        Ok(Self {
            http,
            base_url: region.base_url(),
        })
    }

    /// Absolute URL for a path beginning with `/`.
    pub fn url(&self, path: &str) -> String {
        format!("{}{}", self.base_url, path)
    }

    /// GET a relative path. Internal helper.
    pub(crate) async fn get(&self, path: &str) -> Result<Response, Knowbe4Error> {
        let url = self.url(path);
        self.send_with_retry(|| self.http.get(&url).send()).await
    }

    /// Public escape hatch used by integration tests.
    #[doc(hidden)]
    pub async fn raw_get(&self, path: &str) -> Result<Response, Knowbe4Error> {
        self.get(path).await
    }

    async fn send_with_retry<F, Fut>(&self, make_req: F) -> Result<Response, Knowbe4Error>
    where
        F: Fn() -> Fut,
        Fut: std::future::Future<Output = Result<Response, reqwest::Error>>,
    {
        let mut backoff = 1u64;
        for attempt in 0..=MAX_RETRIES {
            let resp = make_req().await?;
            if resp.status() != 429 || attempt == MAX_RETRIES {
                return Ok(resp);
            }
            sleep(Duration::from_secs(backoff)).await;
            backoff = (backoff * 2).min(MAX_BACKOFF_SECS);
        }
        unreachable!()
    }

    /// GET `path`, following `page`/`per_page=500` pagination until a page
    /// comes back with fewer than `PAGE_SIZE` items. KnowBe4's Reporting API
    /// has no `Link` header, unlike Okta's.
    pub(crate) async fn get_all_pages<T>(&self, path: &str) -> Result<Vec<T>, Knowbe4Error>
    where
        T: DeserializeOwned,
    {
        let sep = if path.contains('?') { '&' } else { '?' };
        let mut all = Vec::new();
        let mut page = 1u32;
        loop {
            let paged_path = format!("{path}{sep}page={page}&per_page={PAGE_SIZE}");
            let resp = self.get(&paged_path).await?;
            if !resp.status().is_success() {
                let status = resp.status().as_u16();
                let message = resp.text().await.unwrap_or_default();
                return Err(Knowbe4Error::Api { status, message });
            }
            let items: Vec<T> = resp.json().await?;
            let got = items.len();
            all.extend(items);
            if got < PAGE_SIZE {
                break;
            }
            page += 1;
        }
        Ok(all)
    }
}
```

use chrono::{DateTime, NaiveDate, Utc};

// Authentication:
//   Authorization: Bearer <api_key>
//
// Base URL: regional (e.g. https://us.api.knowbe4.com). Supplied via the
// `knowbe4_api_key`/`knowbe4_region` config fields or the
// `KNOWBE4_API_KEY`/`KNOWBE4_REGION` env vars.

/// Resolve the effective (start, end) Unix-second window for time-windowed
/// collectors. Falls back to the last 90 days when the caller passes no
/// explicit range, mirroring `OktaSystemLogCollector`.
pub(crate) fn resolve_date_window(dates: Option<(i64, i64)>) -> (i64, i64) {
    match dates {
        Some(d) => d,
        None => {
            let now = Utc::now();
            let start = now - chrono::Duration::days(90);
            (start.timestamp(), now.timestamp())
        }
    }
}

/// Best-effort parse of a KnowBe4 timestamp (RFC 3339 or `YYYY-MM-DD`) into
/// Unix seconds. KnowBe4 does not guarantee one timestamp format across
/// every resource, so both are tried.
fn parse_ts(s: &str) -> Option<i64> {
    if let Ok(dt) = DateTime::parse_from_rfc3339(s) {
        return Some(dt.timestamp());
    }
    NaiveDate::parse_from_str(s, "%Y-%m-%d")
        .ok()
        .and_then(|d| d.and_hms_opt(0, 0, 0))
        .map(|dt| DateTime::<Utc>::from_naive_utc_and_offset(dt, Utc).timestamp())
}

/// True when `s` parses into `[start, end]`, or when it's missing/
/// unparseable — evidence with an unrecognized date shape is kept rather
/// than silently dropped.
pub(crate) fn in_window(s: &Option<String>, start: i64, end: i64) -> bool {
    s.as_deref()
        .and_then(parse_ts)
        .map(|ts| ts >= start && ts <= end)
        .unwrap_or(true)
}

    pub fn knowbe4_api_key_resolved(&self) -> Option<String> {

    /// Resolve KnowBe4 region as a lowercase code (`us`/`eu`/`ca`/`uk`/`de`).
    /// Unrecognized or missing values default to `us`. Returns a `String`
    /// rather than `knowbe4_rs::Knowbe4Region` so this module has no
    /// unconditional dependency on the optional `knowbe4-rs` crate — callers
    /// behind `#[cfg(feature = "knowbe4")]` parse it with `.parse()`.
    pub fn knowbe4_region_resolved(&self) -> String {
        let raw = std::env::var("KNOWBE4_REGION")
            .ok()
            .or_else(|| self.knowbe4_region.clone())
            .unwrap_or_default();
        match raw.trim().to_lowercase().as_str() {
            r @ ("us" | "eu" | "ca" | "uk" | "de") => r.to_string(),
            _ => "us".to_string(),

    }
